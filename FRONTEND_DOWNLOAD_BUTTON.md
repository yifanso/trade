# 前端下载按钮功能说明

## 功能概览

已为前端页面添加了**数据下载按钮和管理界面**，可以直接从Web界面触发QLib数据的下载和更新。

## 新增页面和功能

### 1. 数据下载管理页面 (`/download-manager`)

**访问方式**：
- 通过导航菜单点击 "📥 数据下载管理"
- 或直接访问 `http://localhost:5000/download-manager`

**功能**：
- ✅ **立即触发下载** - 输入股票代码并立即下载
- ✅ **查看下载状态** - 显示定时下载器的运行状态
- ✅ **下载统计信息** - 显示历史下载次数和成功率
- ✅ **下载历史** - 显示最近的下载记录

**界面组成**：
1. **触发下载表单** - 输入一个或多个股票代码
2. **下载状态面板** - 实时显示下载器状态
3. **统计信息面板** - 显示下载统计数据
4. **使用提示** - 说明增量更新等特性

### 2. 增强的下载页面 (`/download`)

**新增内容**：
- ⚡ **快速触发下载区域** - 快速跳转到下载管理器
- 📊 **手动下载区域** - 原有的手动指定时间范围的下载功能

### 3. 更新的仪表板页面 (`/`)

**新增内容**：
- 📥 **数据下载管理** 链接 - 快速访问下载管理器

## API 端点

### POST /api/trigger-download
触发数据下载

**请求**：
```json
{
  "stock_codes": "000858.SZ, 000651.SZ, 600519.SH"
}
```

**响应（成功）**：
```json
{
  "status": "success",
  "message": "成功下载 3/3 只股票",
  "successful": 3,
  "total": 3,
  "details": {
    "000858.SZ": {
      "records": 2500,
      "success": true
    },
    "000651.SZ": {
      "records": 2300,
      "success": true
    },
    "600519.SH": {
      "records": 2400,
      "success": true
    }
  }
}
```

**响应（错误）**：
```json
{
  "status": "error",
  "message": "下载失败: [错误信息]"
}
```

### GET /api/download-status
获取下载器状态和统计信息

**响应**：
```json
{
  "status": "success",
  "running": true,
  "scheduler_active": true,
  "recent_downloads": [
    {
      "stock_code": "000858.SZ",
      "last_date": "2026-04-10",
      "status": "success"
    }
  ],
  "statistics": {
    "unique_stocks": 10,
    "total_downloads": 100,
    "successful": 98,
    "failed": 2,
    "avg_duration_seconds": 1.5,
    "last_download": "2026-04-10T16:00:00"
  }
}
```

## 使用示例

### 示例1: 快速下载单只股票

1. 访问 `/download-manager`
2. 在"股票代码"输入框输入 `000858.SZ`
3. 点击"立即触发下载"按钮
4. 等待下载完成，查看结果

### 示例2: 批量下载多只股票

1. 访问 `/download-manager`
2. 在输入框输入 `000858.SZ, 000651.SZ, 600519.SH`（使用逗号分隔）
3. 点击"立即触发下载"按钮
4. 系统会同时下载这3只股票

### 示例3: 从手动下载页面跳转

1. 访问 `/download`
2. 点击"⚡ 快速下载"按钮
3. 弹出对话框，输入股票代码
4. 自动跳转到下载管理器并填充股票代码

## 前端交互流程

### 触发下载流程

```
用户输入股票代码
    ↓
点击"立即触发下载"按钮
    ↓
禁用按钮，显示加载动画
    ↓
发送 POST /api/trigger-download 请求
    ↓
后端调用 DataDownloader.download_data()
    ↓
后端返回结果
    ↓
显示成功/错误消息
    ↓
显示详细下载结果表格
    ↓
2秒后自动刷新页面
```

## 故障排查

### 问题1: "下载失败: ModuleNotFoundError: No module named 'apscheduler'"

**解决**：
```bash
pip install apscheduler>=3.10.0
```

### 问题2: "页面加载为空"

**检查**：
1. `DataDownloader` 是否安装正确
2. 缓存目录是否有权限写入
3. QLib 数据路径是否正确

### 问题3: "按钮点击无反应"

**检查**：
1. 浏览器控制台是否有错误
2. Flask 后端是否正常运行
3. API 端点是否正确
