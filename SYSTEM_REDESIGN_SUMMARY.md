## 🎯 核心功能清单

### 功能1️⃣: 下载历史交易数据 ✅

**需求**: "可以从qlib主动定期下载更新交易数据的功能"

**实现现状**:
- ✅ DataDownloader类 (600+行)
  - 增量更新逻辑
  - 后台定时任务 (APScheduler)
  - SQLite历史记录
  - 错误重试机制

- ✅ 使用示例1: 一次性下载
  ```python
  from qlib_backtest.data.downloader import DataDownloader
  downloader = DataDownloader()
  downloader.download_data(['000858.SZ'], incremental=True)
  ```

- ✅ 使用示例2: 定时自动下载
  ```python
  downloader.start_scheduler(
      stock_codes=['000858.SZ'],
      cron_expression='0 2 * * *'  # 每日凌晨2点
  )
  ```

- ✅ 使用示例3: 配置化方案
  ```yaml
  # config/default_config.yaml
  stocks:
    - 000858.SZ
    - 000651.SZ
  ```

**存储位置**: `~/.cache/qlib_backtest/data/*.csv`

**支持的功能**:
- 自动检测缺失数据
- 避免重复下载
- 记录下载历史
- 自动重试失败

## 📞 技术支持

### 常见问题

**Q1: 数据下载很慢怎么办?**
- A: 首次全量下载正常需要几秒，后续增量更新<1秒。若需加速可并行下载 (见高级优化)。

**Q2: 回测为什么都是0收益?**
- A: 可能是a)信号过少/过多，b)策略参数不适合当前数据，c)佣金滑点太高。建议: 查看信号数量，调整参数，或降低成本设置。

**Q3: 如何自定义策略?**
- A: 参考 QUICK_START_GUIDE.md 中的"如何自定义策略"章节，实现3个方法即可。

**Q4: 支持多资产回测吗?**
- A: 目前支持A股多只股票。期货/加密/外汇支持需通过扩展接口实现 (见扩展指南)。

**Q5: 生产环境如何部署?**
- A: 参考 ARCHITECTURE_GUIDE.md 第8章 "运维监控部署"，使用Docker + Kubernetes。
